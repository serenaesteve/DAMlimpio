Enunciado de la actividad

Módulo: Entornos de Desarrollo
Curso: 1º Desarrollo de Aplicaciones Multiplataforma (DAM)
Evaluación: Segunda evaluación

Contexto

En esta actividad se trabajará el desarrollo de una pequeña aplicación escribiendo el código directamente desde un editor de texto. El objetivo es comprender el proceso de creación, organización y ejecución de un proyecto software sin depender de entornos de desarrollo integrados, utilizando herramientas básicas del sistema.

Tareas a realizar
-Editor de código
-Utilizar gedit como editor de texto para escribir el código.

Configurar gedit con:
-Resaltado de sintaxis del lenguaje utilizado (Python o PHP).
-Guardado correcto de los archivos con su extensión correspondiente (.py o .php).
Estructura del proyecto

Crear una carpeta para el proyecto con una estructura sencilla, por ejemplo:

proyecto/
├── src/
├── docs/
└── README.txt


Guardar el código principal dentro de la carpeta src.

Ejecución del programa
-Ejecutar el programa desde la terminal, no desde el editor.
-Python: python3 nombre_archivo.py
-PHP: php nombre_archivo.php
-Comprobar que el programa funciona correctamente.

Control de versiones (básico)
-Inicializar un repositorio Git desde la terminal.
-Realizar al menos 3 commits durante el desarrollo.
-Comentar brevemente qué cambios se han hecho en cada commit.

Documentación
Crear un archivo README.txt donde se indique:
-Lenguaje utilizado.
-Cómo ejecutar el programa.
-Breve explicación de qué hace el programa.
-Dificultades encontradas durante el desarrollo.

Entrega

El alumnado deberá entregar:
-Carpeta del proyecto comprimida o repositorio Git.
-Archivo README.txt.
-(Opcional) capturas del programa ejecutándose en la terminal.
-Criterios de evaluación
-Uso correcto de gedit como editor.
-Organización básica del proyecto.
-Ejecución correcta desde la terminal.
-Uso básico de Git.
-Claridad de la documentación.
