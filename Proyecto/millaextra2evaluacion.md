Enunciado del Proyecto Intermodular

Ciclo: Desarrollo de Aplicaciones Multiplataforma (DAM)
Curso: 1º
Módulo: Proyecto Intermodular

Título del proyecto

Gestión básica de un portfolio fotográfico

Descripción del proyecto

El proyecto consiste en el desarrollo de una aplicación sencilla que permita organizar y mostrar el trabajo fotográfico de una fotógrafa.
La aplicación se centrará en la gestión de sesiones y fotografías, integrando los contenidos vistos en los distintos módulos del curso.

El proyecto se realizará de forma individual y utilizando herramientas básicas de desarrollo.

1. Funcionalidades obligatorias

La aplicación deberá permitir:
-Gestión de sesiones
-Añadir una sesión (cliente, fecha, tipo y precio).
-Listar todas las sesiones.
-Buscar una sesión por identificador.

2. Gestión de fotografías
-Asociar varias fotografías a una sesión.
-Listar las fotografías de una sesión concreta.

3. Persistencia de datos
-Almacenar la información en una base de datos relacional.
-Utilizar claves primarias y foráneas correctamente.

Ejecución
-La aplicación se ejecutará por consola (Python o PHP).
-El código se escribirá usando gedit y se ejecutará desde la terminal.

Integración por módulos
-Programación:
Desarrollo del menú, funciones y control de errores.

-Bases de Datos:
Diseño del modelo entidad–relación y creación de la base de datos.

-Sistemas Informáticos:
Organización de carpetas, permisos básicos y copias de seguridad del proyecto.

-Entornos de Desarrollo:
Uso de gedit, estructura del proyecto y documentación básica.




Cómo encajan los ejercicios anteriores en el proyecto intermodular

Bases de Datos
➜ Usas el mismo diseño que hiciste en la actividad de BD:

-Tablas de sesiones, clientes y fotografías.
-Las consultas que ya hiciste te sirven aquí.

Programación (Python o PHP)
➜ El programa de consola que hiciste:

-Menú.
-Alta de sesiones.
-Búsqueda.
-Cálculo de ingresos.
-Solo tienes que conectarlo a la base de datos en vez de usar listas.

Sistemas Informáticos
➜ Reutilizas:

-La estructura de carpetas.
-Usuarios y permisos.
-Copias de seguridad del proyecto o de la base de datos.

Entornos de Desarrollo
➜ Ya cumples:
-Uso de gedit.
-Ejecución desde terminal.
-Organización del proyecto.
-README y documentación.
